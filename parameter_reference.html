config.py sim change this file first — everything reads from it
parameterdefaultwhat it controls
ARENA_W / ARENA_H800Room size in pixels. 1 px ≈ 1 cm at your scale.
ROBOT_RADIUS12Physical body size. Affects collision detection and sensor ray origins.
ROBOT_MAX_SPEED3.0Motor speed ceiling. Must stay above intruder speed to allow capture.
ROBOT_START(100, 400)Robot spawn position each trial. Change to test different starting sides.
N_PROX_SENSORS8Number of proximity rays. 8 = 45° spacing (e-puck configuration).
PROX_RANGE80Max obstacle detection distance. Reduce for short-range IR, raise for LIDAR.
PROX_GAIN1.5Fixed multiplier on avoidance channel. Raise = stronger obstacle aversion.
TARGET_SENSOR_FOV1.8 radHalf-angle of target sensor. ±1.8 rad = ~206° total field of view.
TARGET_SENSOR_RANGE900Max intruder detection distance. 900 = full arena coverage.
TARGET_GAIN2.5Fixed multiplier on pursuit channel. Raise = faster approach when target is visible.
PURSUIT_WEIGHT α1.0Baseline pursuit aggressiveness. RL policy adjusts this at runtime.
AVOIDANCE_WEIGHT β1.2Baseline avoidance strength. β > α prioritises not hitting obstacles.
FORWARD_BIAS0.4Stage 3 only. Constant drive added so robot never fully stops. Set 0 to disable.
MOTOR_FLOOR0.2Stage 3 only. Minimum motor speed after avoidance subtraction.
STUCK_PATIENCE80Stage 3 only. Steps before declaring the robot stuck. Raise = slower to react.
STUCK_DIST_THRESH8 pxStage 3 only. Movement threshold for stuck detection. Raise = more sensitive.
RECOVERY_DURATION40Stage 3 only. Steps of reverse-turn before resuming normal control.
CAPTURE_RADIUS20How close the robot must get to count as a capture.
INTRUDER_RADIUS10Intruder body size. Affects capture detection and obstacle bounce.
INTRUDER_SPEED1.8Intruder constant velocity. Keep below ROBOT_MAX_SPEED.
INTRUDER_START(700, 400)Intruder spawn. Opposite end of arena from robot by default.
run_experiments.py experiment controls Stage 1 batch runs — also imported by Stages 2 and 3
parameterdefaultwhat it controls
DENSITY_LEVELS[0%, 5%…40%]Which obstacle densities to test. Add or remove values to change the sweep.
N_LAYOUTS4Fixed obstacle arrangements per density. More = better statistical coverage.
N_TRIALS5Independent trials per layout (different intruder heading each time).
MAX_STEPS4000Episode timeout. Uncaptured trials get this value as their capture_step.
BASE_SEED42Root RNG seed. Change to get a completely different set of layouts.
OBS_RADIUS_MIN/MAX15 / 30Obstacle size range in pixels. Represents columns (small) to furniture clusters (large).
MIN_CLEARANCE_ROBOT60Keeps obstacles away from the robot spawn. Prevents instant collision at trial start.
MIN_CLEARANCE_INTRUDER50Keeps obstacles away from the intruder spawn.
train_rl.py RL REINFORCE training — tune these if learning is unstable
parameterdefaultwhat it controls
N_EPISODES2000Total training episodes. For a weak laptop use 500.
MAX_STEPS_EP2000Episode length cap during training. Shorter = faster iteration but less signal.
LR_ACTOR5e-4Actor learning rate. Lower if policy oscillates; raise if learning is very slow.
LR_CRITIC1e-3Critic learning rate. Usually 2–5× actor rate.
LR_LOG_STD1e-4Exploration decay rate. Lower = policy stays exploratory longer.
GAMMA0.99Discount factor. 0.99 = cares about rewards up to ~100 steps ahead.
ENTROPY_COEF0.01Exploration bonus weight. Raise if policy collapses to one behaviour too early.
GRAD_CLIP1.0Max gradient norm per update. Prevents large destabilising steps.
CURRICULUM_END500Episodes before full density range is introduced. Raise if early learning is erratic.
EVAL_INTERVAL100How often to print and log evaluation capture rate. Lower = more verbose.
SAVE_INTERVAL500How often to save a policy checkpoint (.npz file).
DENSITY_EASY[0%, 5%, 10%]Densities used during curriculum warmup phase.
DENSITY_FULL[0%…40%]Full density range used after curriculum phase ends.
rl_env.py RL reward shaping — change if robot learns the wrong behaviour
parameterdefaultwhat it controls
LAMBDA_DIST0.01Per-step penalty for distance to intruder. Encourages continuous approach.
LAMBDA_COLL0.5Per-step penalty for collision. Raise if robot learns to ram through obstacles.
LAMBDA_TIME0.002Small per-step time cost. Encourages fast capture over slow cautious approach.
LAMBDA_CAPT10.0Terminal bonus on capture. Must dominate all per-step penalties summed over episode.
MAX_STEPS4000Training episode timeout. Matches experiment runner timeout.
DENSITY_RANGE(0.0, 0.35)Density range sampled during training when no explicit density is set.
rl_policy.py RL network architecture — change before training, not after
parameterdefaultwhat it controls
HIDDEN32Hidden layer size. Raise to 64 for more expressive policy; raises training time.
DELTA_MAX0.8Max weight adjustment output. Policy can shift α or β by at most ±0.8 from baseline.

parameterdefaultwhat it controls

BETA_MIN / BETA_MAX0.1 / 2.5Hard clamp on effective β. Prevents the policy from disabling avoidance entirely.
